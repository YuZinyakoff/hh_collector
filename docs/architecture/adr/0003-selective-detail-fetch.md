# ADR-0003: Полные detail-карточки запрашиваются выборочно

## Статус
Accepted

## Контекст
Тотальная повторная детализация всех активных вакансий на каждом цикле слишком дорога и не обязательна для MVP.

## Решение
Detail fetch выполняется только для:
- новых вакансий;
- вакансий с изменившимся short payload;
- вакансий без актуального detail snapshot;
- вакансий, достигших TTL на refresh;
- ручного operator refetch.

## Последствия
Плюсы:
- ниже нагрузка на API и систему;
- лучше соотношение ценности и стоимости сбора.

Минусы:
- часть изменений detail-содержимого между sweep может остаться ненаблюдаемой.
